# Returns

## Purpose
Separate return documents linked to original sale or order, with line-level returned quantities and refund totals.

## Why this feature exists
The project explicitly models reverse flow as controlled documents instead of negative sale rows.

## Scope
- initiate return
- receive and inspect items
- approve and refund return
