# Stocktakes

## Purpose
Stock count sessions that compare expected system quantity with physically counted quantity and post gain/loss adjustments.

## Why this feature exists
Retail operations need controlled stock reconciliation without directly editing balances.

## Scope
- open count session
- capture line counts
- post variance adjustments
